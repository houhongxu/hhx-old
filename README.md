## 规范

### gitignore

每个包都应该有独立 gitignore，以免有文件疏漏

### 提交信息规范

采用比较多的是 Angular 规范

格式为：
`<type>(<scope>): <subject>`

不在具体包内则为 all

type 则依据 Angular 规范微调

- feat 实现新功能
- fix 修复问题
- chore 无法归类的杂项
- refactor 重构
- test 测试
- perf 性能优化
- doc 文档
